Explanation of Node.js

Node.js is runs for server-side and networking applications. Node.js applications are written in JavaScript, They run through Node.js runtime on multiple operating systems such as Microsoft Windows and Linux.

Node changing the server side languages

JavaScript has been on the client-side and make the Browser doing the processing information. Node.js change the server by make the processing can be done with JavaScript as well.

Both client-side and sever-side using JavaScript the average programmer needn't know two different scripting languages to develop a great web application. Does awesome non blocking IO making processing much faster for a web sever and uses less memory.

Awesome non blocking IO

We will explain non blocking IO through an example, you can read multiple files at the same time without waiting on something or blocking. Normally you would have to read the files one after the other waiting on each file to finish reading, before moving onto the next or blocking that can be very slow.


Diagram depicts how Node.js server works. At a high level, Node.js server has two parts to it:
  • At the front, you have Chrome V8 engine (single threaded), event loop and other C/C++ libraries that run your JS code and listen to HTTP/TCP requests.
  • And at the back of the server, you have libuv (includes libio) and other C/C++ libraries that provide asynchronous I/O.
